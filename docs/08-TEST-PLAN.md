# План тестирования

**Проект:** JRNL-2026 | **Версия:** 1.0 | **Дата:** 2026-03-23

---

## 1. Подход к тестированию

- **Тип**: ручное функциональное тестирование
- **Среда**: staging-сервер (идентичный production)
- **Браузеры**: Chrome, Firefox, Safari, Edge (последние версии)
- **Устройства**: десктоп (1920×1080, 1366×768), планшет (768×1024), мобильный (375×667)
- **Тестировщик**: разработчик ЦИИ + представитель заказчика (приемочное)

---

## 2. Тест-кейсы

### TC-01: Главная страница

| Шаг | Действие | Ожидаемый результат | Статус |
|-----|---------|---------------------|--------|
| 1 | Открыть главную страницу | Страница загружается за ≤3 сек | ⬜ |
| 2 | Проверить шапку | Логотип, название, меню, поиск, языки, вход, ISSN — отображаются | ⬜ |
| 3 | Проверить подвал | Эмблема, учредитель, адрес, год основания, метрика — отображаются | ⬜ |
| 4 | Проверить блок текущего номера | Название выпуска, обложка (если есть), список статей | ⬜ |
| 5 | Проверить кнопки CTA | «ПОДАТЬ СТАТЬЮ», «ТЕКУЩИЙ НОМЕР» — кликабельны, ведут куда нужно | ⬜ |
| 6 | Проверить HTTPS | Замок в адресной строке, нет mixed content | ⬜ |

## 3. Критерии прохождения

- **Блокирующие**: TC-01, TC-06, TC-08, TC-09, TC-10, TC-11, TC-14 (шаги 1–3) — все шаги пройдены
- **Важные**: TC-02–TC-07, TC-12, TC-13, TC-15 — ≥90% шагов пройдены
- **Желательные**: TC-14 (шаги 4–6) — пройдены при наличии времени
