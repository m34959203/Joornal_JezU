# Правила работы с OJS

## Версия
- OJS 3.4.x (latest stable) — НЕ использовать RC/beta
- Проверять changelog перед обновлением

## Золотое правило
**НИКОГДА не модифицировать ядро OJS.** Все изменения — через:
1. Child theme (дизайн, шаблоны)
2. Плагины (функционал)
3. Локали (переводы)
4. Конфигурация (config.inc.php, админка)

## Child Theme
- Наследует от `defaultthemeplugin`
- Директория: `plugins/themes/zhezujournal/`
- Переопределять только те шаблоны, которые нужно изменить
- Остальные наследуются автоматически
- При обновлении OJS — тестировать совместимость темы

## Плагины
- Устанавливать через Plugin Gallery (админка) где возможно
- Кастомные плагины — в `plugins/generic/` или `plugins/themes/`
- Каждый плагин — свой `version.xml`
- Тестировать плагины после обновления OJS

## Обязательные плагины
| Плагин | Назначение |
|--------|-----------|
| DOI | Присвоение DOI статьям |
| CrossRef XML Export | Экспорт метаданных в CrossRef |
| ORCID Profile | Интеграция с ORCID |
| Usage Statistics | Статистика просмотров/загрузок |
| Google Analytics / Custom Header | Яндекс.Метрика |
| Backup | Или скрипт бэкапа |

## Workflow рецензирования
- Тип: двойное слепое (по требованию ККСОН)
- Этапы: Submission → Review → Copyediting → Production → Publication
- Email-уведомления на каждом этапе
- Форма рецензирования — кастомная (оценки + комментарии)

## Файлы
- Рукописи: `/var/www/files/` (вне webroot!)
- Публичные файлы: `/var/www/html/public/`
- Максимальный размер загрузки: 50 MB
- Форматы: .doc, .docx, .tex, .pdf

## Cron
```bash
# Каждые 4 часа — обработка задач OJS
0 */4 * * * php /var/www/html/tools/runScheduledTasks.php
```
