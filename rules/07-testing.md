# Правила тестирования

## Среды
| Среда | URL | Назначение |
|-------|-----|-----------|
| dev | localhost:8080 | Разработка (docker-compose.dev) |
| staging | staging.journal.zhezu.edu.kz | Тестирование, демо заказчику |
| production | journal.zhezu.edu.kz | Боевой сервер |

## Правило: НЕ тестировать на production
- Все тесты — на staging
- Production-деплой только после прохождения всех тестов на staging
- Тестовые пользователи — с пометкой `[TEST]` в имени

## Виды тестирования
1. **Smoke-тест** — после каждого деплоя на staging (главная грузится, вход работает)
2. **Функциональное** — по тест-кейсам из `docs/08-TEST-PLAN.md`
3. **Кроссбраузерное** — Chrome, Firefox, Safari, Edge
4. **Адаптивное** — 4 разрешения (375, 768, 1366, 1920)
5. **Безопасность** — SQL-инъекции, XSS, доступ к конфигам
6. **Производительность** — Lighthouse, время загрузки

## Чек-лист smoke-теста (после каждого деплоя)
- [ ] Главная страница грузится (HTTPS, < 5 сек)
- [ ] Меню работает (все пункты кликабельны)
- [ ] Переключение языков (ru → kk → en)
- [ ] Вход в систему (тестовый аккаунт)
- [ ] Архив — номера отображаются
- [ ] Поиск — возвращает результаты
- [ ] Страница статьи — PDF доступен

## Отчетность
- Баги фиксировать в GitHub Issues с тегами:
  - `bug/critical` — блокирует работу
  - `bug/major` — влияет на функционал
  - `bug/minor` — косметика
  - `bug/security` — уязвимость
